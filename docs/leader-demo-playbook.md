# 领导演示测试用例与模拟数据

## 1. 演示目标

本次演示重点不是“功能点堆砌”，而是让领导看到一条完整闭环：

1. 从掌厅首页进入智能体广场
2. 进入规则配置助手
3. 通过自然语言或结构化表单创建规则
4. 规则创建后，在“我的规则”可查看详情与下一步动作
5. 提醒消息自动进入“消息中心”
6. 对订单异常类提醒，可继续跳转到业务排障助手处理

建议整体演示时长控制在 `8-12 分钟`。

## 4. 领导演示主线

## 4.1 主线 A：规则创建闭环

### 用例 A1：从首页进入智能体广场

- 目的：展示和现有掌厅入口融合，不破坏原首页
- 入口：掌厅首页
- 操作：
  - 展示原掌厅首页
  - 点击右侧 `你的小移`
- 预期结果：
  - 进入智能体广场
  - 广场采用移动端适配布局

### 用例 A2：进入规则配置助手

- 目的：展示智能体广场到业务助手的承接
- 入口：智能体广场
- 操作：
  - 点击 `规则配置类助手`
- 预期结果：
  - 进入规则配置助手
  - 左侧有可收起的对话记录
  - 页面以聊天交互为主

### 用例 A3：自然语言发起预警规则

- 目的：展示“自然语言识别 + 结构化配置”
- 输入话术：
  - `我想给华星集团配置红名单到期提醒，到期前3天提醒`
- 操作：
  - 在规则配置助手输入框中输入上述文本
  - 点击发送
- 预期结果：
  - 系统识别为 `预警提醒配置`
  - 自动打开配置卡
  - 自动带出：
    - 监控对象：`华星集团`
    - 预警类型：`红名单到期`
    - 提前提醒天数：`3`

### 用例 A4：结构化补全并生成摘要

- 目的：展示规则卡片配置能力
- 前置条件：已执行 A3
- 操作：
  - 若字段未补齐，则补齐：
    - 规则名称：`华星集团红名单到期提醒`
    - 生效周期：`长期有效`
  - 点击 `生成摘要`
- 预期结果：
  - 进入规则摘要页
  - 可见摘要信息：
    - 规则类型
    - 监控对象
    - 预警类型
    - 提醒时间：`到期前 3 天提醒`
    - 生效周期

### 用例 A5：校验并创建规则

- 目的：展示规则创建闭环
- 前置条件：已执行 A4
- 操作：
  - 点击 `确认创建`
- 预期结果：
  - 展示校验过程
  - 展示创建成功结果
  - 系统生成规则编号

## 7. 建议演示话术

### 开场

`这次我们不是单点做一个助手，而是把它嵌入到现有掌厅体系里。领导现在看到的首页入口基本保持原状，我们只增加了一个轻量入口，不打断原有工作台使用习惯。`

### 进入广场

`点击右侧“小移”后，进入的是智能体广场。这里面承载的是我们面向不同业务场景的助手能力。`

### 规则配置主讲

`我先演示一个最典型的场景：客户经理想给集团配置红名单到期提醒。这里既支持自然语言，也支持结构化配置卡。`

### 消息中心主讲

`规则不是配置完就结束了，后续真正触发的提醒会统一进入消息中心，形成可跟踪、可闭环的运营入口。`

### 联动主讲

`如果触发的是订单执行异常，不会停留在提醒层，而是可以直接跳转到排障助手继续处理，形成从发现问题到处置问题的闭环。`

## 10. 现场应对建议

- 如果识别不够理想：切回结构化卡片，手动补字段
- 如果出现不支持分流：解释为系统具备助手路由能力，然后换回标准演示话术
- 如果消息状态不明显：进入消息中心点一次 `标记跟进`
- 如果担心网络或刷新问题：优先走规则配置主线 A + 消息中心主线 B，这两条最稳
